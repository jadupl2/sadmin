Overview of the 'sa.start' and 'sa.stop' functions in Bash Library.                                   .                                
====================================================================================================Bash Library Function

   'start()' Bash Library Function 

   The 'start()' function basically initialize the SADMIN environment.\n
   When you call 'sadm_start()', it will only come back to caller if everything went OK.
   Otherwise it will advise the user of the error and exit(1).

  Here is a summary of the different things it does :
    1) Make sure all $SADMIN directories & sub-dir. exist and have proper permissions.
    2) If $SADN_LOG_APPEND='N', create new log & error log, else append to actual log.
    3) If $SADM_LOG_HEADER='Y', write the log header.
    4) if $SADM_ROOT_ONLY='Y', and current user is not root, show error message and exit(1).
    5) If $SADM_SERVER_ONLY='Y', and not on the SADMIN server, show error message and exit(1).
    6) If $SADM_SADMGRP_ONLY='Y', and the user is not part of the SADM_GROUP (unless 'root'), 
       show error message and exit(1).
    7) If $SADM_USE_DB='Y' but not on SADMIN Server, show error message and exit(1).
    8) If $SADM_USE_RCH='Y', write starting time to the RCH file (With a code 2=Running).
    9) If system is lock (and running on the SADMIN), issue message and exit(1)
   10) If PID file '$SADM_PID_FILE' exist and '$SADM_MULTIPLE_EXEC='Y'', continue normal execution
       If PID file exist and execution time (sec) is less than the '$SADM_PID_TIMEOUT', show error message and exit(1).
       If PID file exist and execution time (sec) exceed the '$SADM_PID_TIMEOUT' a new '$SADM_PID_FILE' is created
       and execution is resume.
 
   *** If any unrecoverable error occurs while executing this function the program is aborted exit(1). ***



    'stop(exit_code)' Bash Library Function 
        - exit_code (0=Success, 1=Error): Variable defined in SADMIN section.

  This should be the one of the last function called at the end of your program.

  What this function does:
    1) Calculate execution Time.
    2) If N='Y', update the rch file (End Time & Elapse Time ...).
    3) Validate the alert group, 
    4) If Y='Y', write the log footer.
    5) If the error log file is empty, then delete it.
    6) Delete the PID file (/opt/sadmin/tmp/sadmlib_std_demo.pid) of the program.
    7) Close log and error log files.
    8) If 500 is not zero, trim the log according to user choice in '500'.
    9) If 30 is not zero, trim the 'rch' according to user choice in 'SADM_MAX_RCLINE'.
   10) Set permission and owner/group to log and rch files.
   11) If on the SADMIN server, then rch and log are immediatly web central directory.
